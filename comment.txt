页面和功能定义
第一版 demo 先帮我做出来一个网页版的 Microsoft To-do，支持以下几个功能：
1. Goal: 支持目标新增、删除、修改、完成（或者目标 curation？）
  需求假设：大目标变具体计划（如难目标细化为 milestone）
问题：Goal 是 tracking 还是 Guiding? @罗泽铭 根据我们的假设，我先做guiding 了
如果是 guiding，AI 的角色是 generate 合适的 subtasks
如果是 tracking，AI 的角色是提醒和监督
  第一步：Goal 输入
  - 用户直接输入
  - 选填：时间 / 衡量标准 / 需要 AI 帮忙拆解
    如
    - 「我已经有计划了」→ 允许直接填 milestone
    - 「我还不确定，需要建议」→ 进入 AI 生成模式
  第二步：AI 提问（仅在必要时）
  - Goal 明确 → 直接生成初步计划
  - Goal 模糊 → 只问 1-2 个关键问题，再生成计划
  第三步：AI 生成任务
  - 用户可以调整 AI 生成的内容
  - 任务列表是可编辑的，而不是被 AI 锁死
  第四步：时间安排
  - 可选：「让我自己安排」or「AI 帮我找空闲时间」
  - AI 结合用户习惯，建议时间安排（可拖拽修改）

Freestyle 架构设计
  1. 组件结构：
    - EnhancedGoalList.tsx - 主组件，展示现有目标列表
    - GoalCreationFlow.tsx - 多步骤目标创建流程组件
    - AITaskGenerator.tsx - AI 任务生成和提问组件
    - TimeScheduler.tsx - 时间安排组件
    - EditableTaskList.tsx - 可编辑任务列表组件
  2. 状态管理：
    - 扩展现有的 Zustand store，添加 AI 相关的状态和操作
    - 创建新的状态层来管理创建流程
  3. 数据流：
    - 用户输入 → AI 轻提示 → 可选 AI 提问 → 任务生成 → 时间安排 → 保存目标
2. Tasks 任务：支持新增、删除、修改（feedback）、完成；单个任务点击后有卡片，可以语音/文字互动
填充时间（有特定时间但目标模糊时提供建议）


可以填写描述、设置截止日期、有一个今天完成的按钮快速将任务加入今天的任务列表，还有一个专门的区域可以通过文字填写任务反馈

=======================================================
已完成的功能和优化（2024年7月更新）
=======================================================

1. Goal 组件优化与 UI 改进
   - 优化侧边栏导航：将 "My Goals" 选项移至侧边栏第二位置，紧跟 "Today"，提高访问便捷性
   - 删除手动拖拽进度条：使用 "Mark as Complete" 按钮替代，优化操作流程与界面简洁度
   - 改进目标卡片样式：完成的目标使用绿色背景与徽章提供视觉反馈
   - 添加目标编辑模态框：完善编辑功能，支持修改标题、描述、类别和任务列表
   - 任务完成交互升级：点击目标任务前的复选框可自动更新目标进度

2. Goal 与 Task 数据同步机制
   - 实现双向同步：从 Goal 创建的任务自动同步到 All Tasks 列表，保持数据一致性
   - 智能状态同步：标记目标为完成时，自动将关联任务标记为完成
   - 删除联动：删除目标时同步删除关联任务
   - Task 修改反向同步：在 All Tasks 中修改任务状态，自动更新关联目标的进度
   - 添加提示信息：在编辑模态框中展示同步状态提示

3. 日期与时间处理增强
   - 时间描述转换功能：创建 dateUtils 工具，将文字描述（如"本周内"、"下周五"）自动转换为具体日期
   - AI 生成任务时间优化：改进 AI 提示，要求生成更具体的任务时间描述
   - 任务日期显示优化：根据日期与当前时间关系智能显示"今天"、"明天"、"周五"或具体日期
   - 高级日期选择器：实现日历视图的日期选择器，支持月份导航和日期网格显示
   - 快捷日期选项：提供"今天"、"明天"、"下周"等快捷日期选择

4. 任务详情页面优化
   - 完整重构任务详情组件：美观易用的日期选择器、优化表单布局
   - 交互体验优化：点击外部自动关闭选择器，改进表单控件布局
   - 用户反馈区域优化：简化反馈提交流程，改进反馈历史显示
   - 优先级选择器：改进了优先级选择的视觉设计
   - 目标关联功能：简化任务与目标的关联流程

5. 调试与性能改进
   - 添加日志系统：在关键操作点添加详细日志，便于开发调试
   - 错误处理优化：捕获并处理 AI 服务调用异常
   - 提升状态管理稳定性：避免引用未定义对象，增加安全检查
   - 生成随机 ID：改进 ID 生成机制，降低冲突可能性
   - 补充代码注释：添加中文与英文注释说明代码功能与逻辑

6. 辅助功能与无障碍优化
   - 添加 ARIA 属性：为表单控件添加 aria-label 与 title 属性
   - 键盘导航支持：优化按钮与表单控件的键盘访问
   - 提升表单元素可访问性：确保所有输入框有适当标签
   - 视觉反馈增强：任务与目标状态变化提供清晰的视觉反馈
   - 色彩对比度优化：确保文本与背景色之间有足够的对比度

7. DeepSeek API 集成优化
   - 完善 API 调用流程：请求与响应处理增强
   - 改进 AI 提示模板：使 AI 生成更实用、具体的任务描述
   - 错误处理机制：添加 fallback 机制处理 API 调用失败情况
   - 模拟数据标识：在使用模拟数据时提供清晰标识
   - 增强 AI 系统提示：指导 AI 生成更具时效性的具体时间建议
